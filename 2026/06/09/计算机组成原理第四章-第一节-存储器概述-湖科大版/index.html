主存的基本结构

我将会告诉你CPU是怎么从告诉内存要读哪个的：

读：

  1. CPU 把目标地址送入 MAR
  2. 地址译码器 “翻译” 地址，选中目标单元
  3. CPU 发 “读” 控制信号
  4. 数据从存储单元 → 数据总线 → MDR
  5. CPU 从 MDR 中读取数据

写：

  1. CPU 把目标地址写入 MAR，发往地址总线，译码选中目标单元。
  2. CPU 把要写入的数据先放到 MDR 里。
  3. CPU 通过控制总线发 “写命令”。
  4. MDR 里的数据通过数据总线，写入被选中的存储单元。

请同学们注意：

  1. 上述两种方式并没有绝对的优劣之分，它们在不同的处理器架构和应用场景中都有各自的适用性和优势。
  2. 除处理器外，大小端方式还涉及外部设备设计、网络数据传输、音视频文件保存等。
  3. 小端与大端方式的区别不仅存在于处理器的寄存器、存储器中，在指令集、系统总线等各个层次中也可能存在差别。
